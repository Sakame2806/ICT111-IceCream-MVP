# Wireframe Specification - IceCream

## Purpose
This document defines the main screens and interface elements for the IceCream clickable prototype.

## Wireframe Scope
The prototype will include the following key screens:
1. Landing page
2. Gallery feed
3. Artwork upload form
4. Artwork detail page
5. Creator dashboard
6. Moderator dashboard

## 1. Landing Page
### Objective
Introduce the platform and guide users to the main actions.

### Layout
- Top navigation bar with logo, search, upload, and profile
- Hero section with headline, short description, and CTA buttons
- Featured artwork section
- Quick access to browse gallery and upload work

### Interactive Elements
- Browse Galleries button
- Upload Portfolio button
- Category chips or links

## 2. Gallery Feed
### Objective
Help users discover artwork by style, fandom, or critique need.

### Layout
- Search bar at the top
- Filter controls for style, fandom, and critique status
- Grid of artwork cards
- Each card includes title, thumbnail, creator name, tags, and critique badge

### Interaction
- Clicking a card opens the artwork detail page
- Filters update the visible gallery list

## 3. Upload Form
### Objective
Allow artists to submit artwork and specify critique needs.

### Layout
- Form fields for title, description, image upload, style tags, fandom tags, and critique focus
- Preview area for uploaded content
- Submit and cancel actions

### Validation Rules
- Title is required
- At least one image is required
- At least one style tag is required
- Critique focus can be optional but recommended

## 4. Artwork Detail Page
### Objective
Show the full artwork and collect structured critique.

### Layout
- Large artwork preview or multi-page viewer
- Artist information and tags
- Critique need badge
- Structured critique list with category labels
- Critique submission form

### Interaction
- Users can switch between pages if a comic or storyboard is uploaded
- Users can submit feedback with a category and comment text

## 5. Creator Dashboard
### Objective
Show creators their post performance and contribution activity.

### Layout
- Summary cards for views, likes, and critique count
- List of uploaded works
- Critique karma indicator
- Optional chart or progress summary

## 6. Moderator Dashboard
### Objective
Support safe and constructive community management.

### Layout
- Reported or flagged content list
- Action buttons for approve, remove, or pin critique
- Quick summary of moderation activity

## Design Notes
- Use a clean, pastel, ice-cream-inspired visual language
- Maintain consistent spacing, typography, and button styles
- Ensure the screens are easy to navigate on mobile and desktop
