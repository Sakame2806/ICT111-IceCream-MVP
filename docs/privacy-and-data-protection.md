# Privacy and Data Protection

## Data Collection Summary
| Data Field | Why It Is Needed | Necessary / Optional | Personal Data? | Sensitive Data? | Visibility Classification | Keep / Remove / Replace | Notes |
|---|---|---|---|---|---|---|---|
| `User_id` | Connects users to uploaded artworks and comments | Necessary | Yes | No | Internal | Keep | Use generated IDs such as `U001`; do not use student ID numbers. |
| `Nickname` | Lets creators and commenters appear with an alias | Necessary | Yes | No | Public | Keep | Use nickname/alias instead of real name. |
| `Password` | Supports login/authentication in prototype | Necessary for authentication, but not for analysis | Yes | Yes | Internal | Replace | Do not store real or plain-text passwords in CSV/mock data. Use Firebase Auth or `ANONYMIZED` placeholder. |
| `Art_id` | Unique identifier for each artwork record | Necessary | No | No | Internal | Keep | Used to connect artwork detail pages and comments. |
| `Artworks.User_id` | Identifies which user uploaded the artwork | Necessary | Yes | No | Internal | Keep | Display creator nickname publicly, not raw user ID. |
| `File_Address` | Stores the artwork image/comic file path or URL | Necessary | Possibly | Possibly | Public when published | Keep with caution | Use only sample, original, or permitted artwork. Avoid private file paths in final public prototype. |
| `Tags` | Enables search/filter by style, fandom, and theme | Necessary | Possibly | No | Public | Keep | Avoid sensitive labels. Tags should be chosen by the creator. |
| `Artworks.Comments` | Links comment IDs to artwork records | Optional | No | No | Internal | Keep if useful | Can be generated by database relation instead of manually stored list. |
| `Artworks.Status` | Shows whether artwork is Published, Underchecking, or Withdrawn | Necessary | No | No | Public / Admin-only for changes | Keep | Users can withdraw own posts; admin can set Underchecking for moderation. |
| `Comments_id` | Unique identifier for each comment | Necessary | No | No | Internal | Keep | Used to manage, display, or moderate comments. |
| `Comments.Art_id` | Links a comment to the related artwork | Necessary | No | No | Internal | Keep | Needed for artwork detail page comment display. |
| `Comments.User_id` | Identifies the user who wrote a comment | Necessary | Yes | No | Internal | Keep | Display reviewer nickname publicly, not raw user ID. |
| `Content` | Stores the critique/comment text | Necessary | Possibly | Possibly | Public when published | Keep with moderation | Comments may include personal, harmful, or offensive content, so moderation is required. |
| `Comments.Status` | Shows whether a comment is Published, Underchecking, or Withdrawn | Necessary | No | No | Public / Admin-only for changes | Keep | Supports comment moderation and user withdrawal. |
| Artwork title | Helps users understand and search artwork | Necessary | No | No | Public | Keep | Required in upload form according to FR-10. |
| Artwork description | Lets creators explain context or critique needs | Optional | Possibly | Possibly | Public | Keep with caution | Remind users not to include private contact details or sensitive personal information. |
| Style/fandom filter selections | Helps users discover relevant artworks | Necessary | Possibly | No | Internal during browsing / Public as artwork tags | Keep | Do not use sensitive identity categories as filter options. |
| Critique focus | Shows what type of feedback the creator wants | Necessary | No | No | Public | Keep | Examples: Anatomy, Coloring/Lighting, Storytelling, Composition. |
| View count / likes / critique count | Supports creator dashboard analytics | Optional | No | No | Internal / creator-only | Keep with caution | Should be visible to the creator, not used as unfair public ranking. |
| Critique karma score | Encourages users to write useful critiques | Optional | Yes | No | Internal / creator-only | Keep with caution | Avoid public pressure or ranking based only on score. |
| Moderation reports / admin notes | Helps moderators handle harmful content | Optional but important for safety | Possibly | Possibly | Admin-only | Keep with restricted access | Only moderators should access report reasons and moderation actions. |

## Privacy Rule for Prototype
IceCream collects only the data needed to support the main MVP workflow: account aliases, artwork uploads, tags, critique comments, artwork/comment statuses, and simple creator analytics. Public screens should display only intended-public information such as nickname, artwork title, artwork image, tags, critique focus, and published comments.

Private or internal fields such as raw user IDs, authentication credentials, moderation notes, and creator-only analytics should not be exposed on public pages. The prototype should use aliases instead of real names or student IDs. Passwords should not be stored in plain text or used for analysis; mock records should use `ANONYMIZED`, and a real implementation should use Firebase Auth or another secure authentication service.

Users should be informed that uploaded artwork and published critique comments may be visible to other users. Upload forms should remind users to share only artwork they own or have permission to use, and comment forms should remind users to give constructive feedback.

## Data Minimization Decision
| Data Field / Practice | Decision | Reason |
|---|---|---|
| Real name | Remove / do not collect | Not needed for artwork sharing or critique. Nickname is enough. |
| Student ID | Remove / do not collect | Creates unnecessary identity and privacy risk. |
| Phone number or private contact details | Remove / do not collect | Not required for MVP workflow. |
| Real password in CSV/mock data | Replace | Passwords are sensitive. Use `ANONYMIZED` in mock data and Firebase Auth for real authentication. |
| Public display of raw `User_id` | Replace with nickname | Raw IDs are internal references and should not be shown to normal users. |
| Private file paths | Replace with safe public/static asset paths | Local/private paths may expose development or user file information. |
| Public analytics ranking | Limit to creator-only dashboard | Avoid unfair pressure or ranking based only on likes, views, or karma. |
| Moderation notes visible to all users | Restrict to admin-only | Reports and admin decisions may contain sensitive or harmful content. |
